PowerZero is a beginner-friendly Raspberry Pi project presented from concept, through development, and into final product.

This Table of Contents provides an overview of each step in the project. Keep in mind that the pages were written contemporaneously with active development, meaning earlier entries may not fully align with the final implementation.

Also note that this is my very first Pi project, and I’m trying to document it for people who might be as new to this as I am. If you’re already experienced, feel free to skip past the earlier pages where I’m just getting started—like setting up my dev environment or screaming at Google trying to figure out why the Pi won't respond to a ping.

The Idea

The idea behind PowerZero. This page outlines the starting plan for the project. Importantly it outlines the framework for decision making:

  1. Objectively Useful: It must solve a real-world problem.
  2. Accessible Simplicity: It must be easy enough for anyone to use without confusion; and,
  3. Achievable Scope: All features must be clearly required in service to usefulness and simplicity.

Part 2: Setting up the Pi

Hello World

I get the Raspberry Pi running (headless) and excute a simple "Hello World" Python script directly on the Pi.

I was having so much fun in this initial step that I decided to create HeadlessZero Bootstrap. Now I can focus on creating PowerZero and future projects without having to spend time on installation and configuration.

Remote Coding

I set up a development environment so that I can get started. I'll make it more robust later, but for now I keep it to the basic requirements:

  1. I develop on my Mac (or PC), not directly on the Pi.
  2. When I make code changes on the Mac, it automagically shows up on the Pi.
    • The code on the Mac is authoritative, not the code on the Pi.
  3. I can access the Pi via SSH, and test the code by running a single Python script.

Part 3: Learning the Components

A Button and an LED

I want to start with the easiest part of the project. Detecting when the user presses a button, and making something change visibly (the LED) as a result.

Time Unplugged

One of the most critical components in this build is the real-time clock (RTC)—a device with its own battery backup to retain time, even during power outages.

Normally, the Raspberry Pi retrieves the current time at boot using the Network Time Protocol (NTP) over an internet connection. However, following the principle of accessible simplicity, I want to eliminate the need for an internet connection—or the end-user hassle of setting one up. To achieve this, the build incorporates a DS3231 RTC module.
